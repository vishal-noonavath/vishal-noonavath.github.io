Code Log 5: Images

Using Images in HTML and CSS

Images can be displayed on webpages either directly in the HTML using the <img> tag or by setting them as "backgrounds" through CSS using the background-image property. Each method serves different purposes in terms of visual design and code structure design.

HTML vs. CSS Image Usage

The <img> tag is used to insert an image directly into the HTML content. It is part of the document structure and can include attributes like src, alt, and width. For example:

<img src="image.jpg" alt="Example image">

In contrast, background-image is used in CSS to apply an image as a decorative or background element. It does not appear in the HTML structure and cannot use attributes like alt, making it less accessible for screen readers. For example:

background-image: url('image.jpg');

HTML images are ideal for content-related visuals like product photos or infographics, while background images are used for design purposes like banners. In both examples, the output is essentially the same, however, the background image doesn't contain an alt attribute. It would be better to use an html image in this particular case since it increases accessibility should the image not load. There would be no indication of a missing image if the background image didn't load.

CSS Properties for Background Images

Beyond background-image, there are several CSS properties to control how background images appear on the webpage. Two common ones are:

  1. background-size - This determines how a background image is scaled within its container. Example: background-size: cover; makes the image cover the entire element area.
  2. background-position - Defines where the image is placed inside the element. Example: background-position: center; centers the image in the element.

Dimensions vs. Resolution

Dimensions refer to an image’s physical width and height in pixels, while resolution describes the image’s level of detail, typically measured in pixels per inch or dots per inch (PPI or DPI). DPI/PPI matter for print but on the web, what matters most is pixel dimensions.

Image File Formats

GIF supports animation and transparency but is limited to 256 colors, making it best for simple graphics or logos.
PNG offers lossless compression and transparency, ideal for high-quality web graphics.
JPG/JPEG uses lossy compression, reducing file size but potentially lowering quality; it’s best for photographs.
WebP combines quality and compression efficiency, providing smaller file sizes with strong visual quality.
SVG is vector-based, meaning it scales without losing quality, making it ideal for icons and logos.
